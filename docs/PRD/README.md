# Product Requirements (PRD)

Product requirements, user stories, and acceptance criteria for ReviewSystem.

## Contents

- `identity-rbac.md` — Identity & RBAC module (Google Sign-In, roles, JWT)
- `phase-1-identity-rbac-implementation.md` — Phase 1 implementation checklist
- `product-overview.md` — Vision, goals, and target users (planned)
- `user-stories.md` — Epics and user stories with acceptance criteria
- `functional-requirements.md` — Feature-level functional requirements
- `non-functional-requirements.md` — Performance, availability, and scalability requirements
- `roadmap.md` — Release phases and milestone planning

## Conventions

- Link user stories to related `BusinessRules/` and `API/` documents.
- Mark requirements with priority: Must / Should / Could.
- Update the PRD when scope changes — record decisions in Architecture ADRs if needed.
