# Architecture Decision Records (ADR)

This directory records significant architectural and technical decisions.

## Format

Each ADR follows this naming convention:

```
NNNN-short-title.md
```

Example: `0001-use-clean-architecture.md`

## ADR Template

```markdown
# NNNN. Title

Date: YYYY-MM-DD
Status: Proposed | Accepted | Deprecated | Superseded

## Context

What is the issue or decision drivers?

## Decision

What is the change being proposed or enacted?

## Consequences

What becomes easier or harder as a result?
```
