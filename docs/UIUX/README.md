# UI/UX

User interface and experience guidelines for the ReviewSystem frontend.

## Contents (planned)

- `design-system.md` — Colors, typography, spacing, and component tokens
- `style-guide.md` — Visual language and branding guidelines
- `wireframes/` — Low-fidelity layout references
- `mockups/` — High-fidelity screen designs
- `accessibility.md` — WCAG compliance and accessibility standards
- `responsive-design.md` — Breakpoints and mobile/tablet/desktop layouts

## Conventions

- Align UI patterns with `src/frontend/src/components/` structure.
- Document new UI components in the design system before implementation.
- Reference `BusinessRules/` for workflow-driven UI behavior.
