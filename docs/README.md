# Documentation

Central documentation hub for the ReviewSystem project.

## Sections

| Directory | Contents |
|-----------|----------|
| `PRD/` | Product requirements, user stories, and acceptance criteria |
| `Architecture/` | System design, diagrams, and technical architecture |
| `Database/` | Data model, ERD, schema, and migration documentation |
| `API/` | REST API specifications, endpoints, and contracts |
| `UIUX/` | UI/UX guidelines, wireframes, and design system |
| `BusinessRules/` | Domain rules, workflows, and business logic definitions |
| `DevOps/` | CI/CD, deployment, environments, and infrastructure |
| `Standards/` | Coding standards, conventions, and quality guidelines |
| `Onboarding/` | New developer setup and team onboarding guide |

## Conventions

- Use Markdown for all documentation unless a specific format is required.
- Update documentation in the same PR as related code changes.
- Keep filenames descriptive and use kebab-case (e.g., `review-workflow.md`).
- Link related documents across sections where appropriate.
