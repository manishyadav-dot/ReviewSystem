# Team Workflow

## Flow

1. Confirm scope (`docs/PRD/`) → 2. Branch from `develop` → 3. Search codebase + read rules → 4. Smallest implementation + tests → 5. PR → 6. Review → 7. Staging → 8. Release

## AI-Assisted Work

Follow `00` generation rules · human reviews all output · author owns merged code · report AI mistakes → update rules

## Definition of Done

Acceptance met · tests updated · docs synced · PR approved · CI green · no scope creep · no unapproved libraries

## Forbidden

Self-merge without review · skip tests · undeclared scope · bypass staging
